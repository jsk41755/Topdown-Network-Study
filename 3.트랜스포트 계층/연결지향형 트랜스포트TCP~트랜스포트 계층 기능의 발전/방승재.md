<details>
<summary>TCP 전이중 방식은 몇개의 소켓일까?</summary>

send 할 때와 recv 할 때는 각각 출발지 ip와 목적지 ip가 반대가 되니까 따로 소켓이 있어야 할 것으로 생각했다. 

그러나 실제로 tuple을 식별하는건 출발지와 목적지가 아닌 local과 remote로 구분된다. 그렇기에 하나의 소켓으로 send와 recv가 가능하다.
즉, 정답은 client와 server 각 1개 씩으로 총 2개가 된다.

</details>

<summary>패킷 손실에 대한 재전송은 어떤 간격으로 하는가?</summary>

간격: 네트워크 왕복 시간인 RTT를 보고 동적으로 계산됩니다.

<details>
<summary>혼잡회피에서 cwnd 값을 줄이는 것을 반복하면 중간중간 네트워크 성능이 저하되는 것 아닌가?</summary>

1. 타임아웃이면 `cwnd = 1 MSS`로 크게 줄임

ACK가 안 와서 **RTO timeout**이 발생하면 TCP는 네트워크 상태가 심각하다고 판단합니다.

이 때는 성능이 일시적으로 감소

2. 중복 ACK로 감지한 손실이면 보통 1까지 안 줄임

TCP Reno 기준으로는 대략:

```go
ssthresh = cwnd / 2
cwnd = ssthresh
```

또는 fast recovery 과정에서 일시적으로 `cwnd`를 조정한 뒤 혼잡회피로 들어갑니다.
그래서 성능 저하가 훨씬 작습니다.

</details>
