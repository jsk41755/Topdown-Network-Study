<details>
<summary>소켓이란 무엇인가?</summary>

**소켓(socket)**은 애플리케이션 프로세스가 네트워크 통신을 하기 위해 OS 커널에 만들어 달라고 요청하는 **통신용 엔드포인트**다.

```go
애플리케이션
  ↓ socket()
OS 커널
  ↓
socket 객체 생성
  ↓
file descriptor 반환
```

file descriptor를 반환하면 socket은 파일인가?
→ 그렇지 않다. 파일에 쓰듯이 사용할 수 있지만 실제로 디스크도 NIC도 아니다.

```go
socker buffer = OS 커널이 RAM 위에 관리하는 네트워크 송수신용 메모리 영역
```

커널 메모리이다.

```go
애플리케이션 메모리
  ↓ write()
커널 메모리의 socket send buffer
  ↓
TCP/IP stack
  ↓
NIC driver / DMA ring
  ↓
NIC
  ↓
네트워크
```

</details>

<details>
<summary>HTTP Method PUT과 PATCH의 차이</summary>

`PUT`은 **리소스 전체를 교체**할 때 사용하고, `PATCH`는 **리소스의 일부만 수정**할 때 사용한다.

사용자 정보가 다음과 같다고 하자.

```go
{
  "id": 1,
  "name": "Kim",
  "age": 20,
  "email": "kim@example.com"
}
```

### PUT 예시

```go
{
  "name": "Lee",
  "age": 21
}
```

서버 구현에 따라 기존 `email`이 유지되지 않고 `null` 처리되거나 삭제될 수 있다.

### PATCH 예시

```go
{
  "age": 21
}
```

의미는:

> `/users/1` 리소스에서 `age`만 21로 바꿔라.

PUT은 일반적으로 멱등하다.
같은 요청은 몇번을 보내든 최종 결과는 같다.

PATCH는 구현에 따라 멱등적일 수도 아닐 수도 있다.

`PATCH /users/1/increase-point`

```go
{
  "amount": 10
}
```

이 요청이 “포인트를 10 증가시켜라”라는 의미라면, 여러번 보낼 때마다 값이 계속 증가한다.

</details>

<summary>HTTP/1.1 은 6개 까지 병렬 TCP 가능하다. 브라우저에서 각 탭마다 6개인가? 전체 6개인가? 요청하는 도메인이 다르면 어떻게 동작하는가?</summary>

참고: https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Connection_management_in_HTTP_1.x

<summary>E-Mail 주소를 가지고 메일 서버 IP를 어떻게 알 수 있는가?</summary>

메일 서버는 `user@example.com` 같은 이메일 주소에서 **`@` 뒤의 도메인**, 즉 `example.com`을 뽑아낸 뒤, **DNS에 MX 레코드**를 조회해서 목적지 메일 서버를 찾는다. SMTP 표준에서도 메일을 전달할 도메인을 식별한 뒤 DNS 조회를 수행하고, 우선 MX 레코드를 찾도록 정의한다.
