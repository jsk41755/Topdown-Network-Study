## 스터디 질문

### 1. IPv4와 IPv6의 차이점은?

<details>
<summary>답변 보기</summary>

| 구분 | IPv4 | IPv6 |
|---|---|---|
| 주소 길이 | 32비트 | 128비트 |
| 주소 개수 | 약 43억 개 | 사실상 고갈 걱정이 거의 없음 |
| 헤더 길이 | 가변 길이, 보통 20바이트 | 고정 길이 40바이트 |
| 헤더 체크섬 유무 | 있음 | 없음 |
| 단편화 처리 | 라우터와 종단 호스트가 수행 가능 | 출발지와 목적지만 수행, 라우터는 단편화하지 않음 |
| 옵션 처리 | 기본 헤더 안에 옵션 필드 존재 | 확장 헤더 방식 사용 |
| 브로드캐스트 | 지원 | 지원하지 않음, 멀티캐스트/애니캐스트 활용 |
| 주소 부족 대응 | NAT를 많이 사용 | 넓은 주소 공간으로 NAT 필요성이 줄어듦 |
| 전환 방식 | 기존 인터넷의 주류 | 듀얼 스택, 터널링 등으로 점진적 도입 |

IPv6는 IPv4의 주소 고갈 문제를 해결하기 위해 주소 길이를 32비트에서 128비트로 확장했다. 또한 라우터의 처리 속도를 높이기 위해 헤더를 고정 길이로 단순화했고, 헤더 체크섬과 라우터 단편화를 제거했다.

IPv6에는 헤더 체크섬이 없다. IPv4에서는 라우터가 TTL 값을 변경할 때마다 헤더 체크섬을 다시 계산해야 했기 때문에 라우터 처리 비용이 증가했다. IPv6는 라우터의 전달 속도를 높이기 위해 헤더 체크섬을 제거했다. 대신 링크 계층과 트랜스포트 계층(TCP/UDP)에서 이미 오류 검사를 수행하므로, IP 계층의 체크섬은 중복으로 보았다.

</details>

### 2. 포워딩과 라우팅은 어떻게 다르고, 왜 데이터 평면과 제어 평면으로 나눌까?

<details>
<summary>답변 보기</summary>

포워딩은 라우터에 도착한 패킷을 어느 출력 포트로 내보낼지 결정해 실제로 이동시키는 동작이다. 매우 빠르게 처리되어야 하므로 보통 하드웨어 수준에서 수행되며 데이터 평면에 해당한다.

라우팅은 출발지에서 목적지까지 어떤 경로로 패킷을 보낼지 계산하는 과정이다. 네트워크 전체의 상태나 라우팅 알고리즘이 필요하므로 상대적으로 느리고 복잡하며 제어 평면에 해당한다.

즉, 라우팅은 길을 정하는 일이고, 포워딩은 정해진 길에 따라 패킷을 넘기는 일이다.

여기서 평면(plane)이라는 표현은 실제 물리적인 평면을 뜻하는 것이 아니라, 네트워크 장비 안의 기능을 역할별로 나눈 논리적 영역을 뜻한다. 같은 종류의 기능들이 하나의 층이나 영역에 놓여 있다고 보고 구분하는 표현이다.

- 데이터 평면: 실제 패킷을 빠르게 처리하고 전달하는 영역
- 제어 평면: 패킷을 어떻게 전달할지 규칙과 경로를 계산하는 영역

이렇게 평면을 나누면 라우터가 하는 일을 “패킷을 직접 처리하는 부분”과 “처리 규칙을 만드는 부분”으로 분리해서 이해할 수 있다. SDN에서는 이 구분이 더 뚜렷해져서, 라우터는 데이터 평면 역할에 집중하고 원격 컨트롤러가 제어 평면 역할을 담당한다.

</details>

### 3. NAT는 왜 필요하고, 어떤 한계가 있을까?

<details>
<summary>답변 보기</summary>

NAT(Network Address Translation)는 사설 네트워크 내부의 여러 장비가 하나의 공인 IP 주소를 공유해서 인터넷에 접속할 수 있게 해주는 기술이다. IPv4 주소가 부족해지면서 가정이나 회사 내부에서는 `10.0.0.0/8`, `192.168.0.0/16` 같은 사설 IP를 쓰고, 외부 인터넷으로 나갈 때 NAT 라우터가 공인 IP와 포트 번호로 변환한다.

NAT 라우터는 내부 호스트의 사설 IP/포트와 외부로 나가는 공인 IP/포트의 대응 관계를 NAT 변환 테이블에 저장한다. 외부 서버가 응답을 보내면 NAT 라우터는 이 테이블을 보고 알맞은 내부 호스트에게 다시 전달한다.

하지만 NAT는 포트 번호를 주소 변환에 사용하기 때문에 계층 구조를 깔끔하게 지키지 못한다. 또한 외부에서 내부 호스트로 먼저 연결을 시작해야 하는 서버, P2P, 일부 실시간 통신에서는 연결 설정이 복잡해질 수 있고 NAT 순회 기술이 필요하다.

</details>
